Detailed Description

Provides locking similar to T:System.Threading.ReaderWriterLockSlim but.

Taken from http://johnculviner.com/achieving-named-lock-locker-functionality-in-c-4-0/ and adapted a bit. Namely:

  • !:GetOrAdd uses a new ReaderWriterLockSlim to overcome possible concurrency issues where the factory delegate could run multiple times.
  • Implemented T:System.IDisposable.
