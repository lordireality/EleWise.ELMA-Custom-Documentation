Detailed Description

Провайдер обратной совместимости при работе с BinaryFile.

Применим только в штатного провайдере! В KB (например, https://www.elma-bpm.ru/KB/article-5499.html) описан вариант добавления версии документа с созданием BinaryFile-а через такой код:

context.Fayl = InterfaceActivator.Create{BinaryFile}(); context.Fayl.Name="Name.docx"; context.Fayl.CreateDate = DateTime.Now; context.Fayl.InitializeContentFilePath(); System.IO.File.Copy("c:\Шаблон1.docx", context.Fayl.ContentFilePath);

Последние две строчки для распределённого провайдера не работают - способ хранения контента изменился, как и способ создания BinaryFile-а. Соответственно изменились и подходы к работе с объектами BinaryFile-ов внутри провайдеров. Однако данное решение наверняка было широко растиражировано и необходимо обеспечить его работоспособность в штатном провайдере. Этот интерфейс как раз для этого.
